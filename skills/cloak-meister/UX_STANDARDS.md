# UX Standards

Review the user goal before visual preference. Prioritize task completion, error prevention and recovery, system status, predictable navigation, clear hierarchy, consistency, and low cognitive load.

## Evidence rules

- Separate visible findings from assumptions and missing states.
- Trace each issue to a screen, control, interaction, component, or supplied behavior.
- Preserve established patterns unless evidence supports redesign.
- Prefer the smallest change that restores clarity or task completion.

## Review order

1. Blocked or destructive workflows
2. Accessibility barriers
3. Validation and recovery
4. Navigation and discoverability
5. Responsive containment
6. Visual hierarchy and consistency
7. Polish after correctness
