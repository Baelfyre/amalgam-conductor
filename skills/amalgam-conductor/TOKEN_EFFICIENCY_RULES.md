# Token Efficiency Rules

Use this file only when a route has overlap or more than three proposed skills.

1. Route the artifact, not every technology mentioned.
2. Give each specialist one distinct output; remove duplicates.
3. Prefer one specialist plus direct validation over a review chain.
4. Load only the matching profile, workflow, and example.
5. Sequence dependent work; do not repeat unchanged context in every prompt.
6. Skip architecture, documentation, security, or QA phases when the requested surface does not require them.
7. Recommend Hidden Dagger without invoking it until the user explicitly chooses the gated pressure test.

Stop trimming only when removing another skill would leave a required artifact, risk, or verification result unowned.
