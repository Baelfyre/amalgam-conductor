# Foundation

The Orchestra follows these rules:

- Align every review with the user's objective and named scope.
- Prefer evidence over complete-looking output; disclose assumptions and missing evidence.
- Use the smallest effective workflow and load supporting files only when needed.
- Assign one specialist to each output and preserve separation of concerns.
- Keep reusable skills local-first until repository tracking is intentional.
- Require factual traceability in documentation and semantic correctness in diagrams.
- Preserve database integrity, UI/UX accessibility, normal QA ownership, and defensive security/privacy boundaries.
- Require explicit authorization and safety gates before controlled destructive testing.
- Stop before production damage, real-user-data exposure, unauthorized access, or irreversible effects.

## External inspiration disclaimer

Ponytail and Caveman are external tools. They are not included, vendored, or required by this repository. Install them separately from their official repositories if desired. The Orchestra adopts only the general principle of clear, focused, and low-filler review behavior.
